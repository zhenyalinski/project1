AAA Testing

Thank you for continuing reading with me today. If you mention someone AAA today, he/she might think of the AAA towing service haha but its totally not it. When it comes to object-oriented language, the AAA testing avoids the spaghetti code situation and keeps your test code more concised, organised and manageable.

But what does AAA really stand for?

Triple AAA testing stands for Arrange-arrange your data, Act- form an action and Assert-asserts that the expectation of the action is correct! With the arrange you will set up the conditions for the test action. Act is when the action will take place on the classtest you are looking to test and arrange for. And last and not least, is assert. You would want to start your act section with the "result=". Assert is what performs the assertion on the actual state of the test you have set up where in our example it will be the calculator class asserting for the addition, subtraction etc.

Why AAA testing is so important today?

AAA testing is very important in the automated testing world today. With AAA you can have a dry run on smaller/separate pieces of your code and see if it runs as intended before it goes into production mode. If we take the arrange, act and assert components of the calculator test, with arrange we are preparing the actual input for the calculator test, when using act we generate a code to test it and assert component verifies that the expected results with the final output that we actually get. Overall, the AAA test definitely saves us time, where we do not need to think of structural options for a test structure and everyone on in a project speaking "same language".
