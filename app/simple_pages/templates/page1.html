Whats Encapsulation?

Encapsulation - is when data is encapsulated an object.Encapsulation protects data like methods and variables within an object to be alternated and keeps them in a separate "department", for example, you have two sharepoint folders that are one for accounting and the other is for underwriting. You would have those folders completely separated from each other where people from accounting only can access the accounting folder and underwriting only has access to the underwriting folders.For example in the calculator_test.py there is a code "calc1 = Calculator() and calc2 = Calculator()" which demonstrates an encapsulation and classes being used independently with independent results to them

Benefits of the inheritance is that it displays real-world relationships,it allows us to reuse the code and it inherits any subclasses related to any classes being inherited. Have you ever heard of method when it comes to python?

Method is a function that will be located within a particular class. It is a function that may operate without a class but once it joins under a class it turns into a method. There are two types if methods, static and instance. When it comes to static method or the word static, you do not instantiate it and will only use it when you need to create just a single instance of lets say, an instance of subtraction and with the instance method will work on each individual project and use the "self" to call for a particular class we work at that moment. What is the purpose of SOLID in the object-oriented programming today?

Before we identify what is the purpose of SOLID and let's first simply identify what SOLID stands for. S - single-responsibility principle, O - open-closed principle, L - liskov substitute principle, I - interface-segregation principle and D is for dependency inversion principle. Overall, SOLID is needed to follow a set of rules and principles in object-oriented design. It has been developed overtime to eliminate as much technical debt as possible and improve testability of your code which could significantly save a lot of time.
